路由和菜单

Elite 为了方便管理和组织成一个关键架构，使用 routers.js 统一管理

/src/router/routes.js

基本结构

在这一部分，脚手架通过结合一些配置文件、基本算法及工具函数，搭建好了路由和菜单的基本框架，主要涉及以下几个模块/功能：

  • 路由管理 - 通过约定的语法在 routers.js 中配置路由
  • 菜单生成 - 根据路由配置来生成菜单。菜单项名称，嵌套路径与路由高度耦合。
  • 面包屑 - 组件中内置的面包屑也可由脚手架提供的配置信息自动生成。

下面简单介绍下各个模块的基本思路，如果你对实现过程不感兴趣，只想了解应该怎么实现相关需求，可以直接查看需求实例。

路由

目前脚手架中所有的路由都通过 routers.js 来统一管理，在配置中我们增加了一些参数，如 label,icon，来辅助生成菜单。其中：

  • label 和 icon 分别代表生成的菜单项的文本和图标
  • path - 当前菜单点击时候的完整路由，例如：/dashboard/welcome
  • auth - 可以访问这个页面的权限名称，数据类型是 Array

菜单

菜单根据 routers.js 生成，具体逻辑在 src/components/menu/menus.js 中的 formatter 方法实现。

面包屑

面包屑根据 routers.js 生成，具体逻辑在 src/components/breadCramb/breadCramb.js 中的 formatter 方法实现。

需求实例

一级菜单


{
  icon: 'el-icon-menu',
  label: '工作台',
  showInMenu: true,
  path: '/dashboard/welcome',
  auth: ['admin', 'customer'],
  name: 'Welcome',
  component: () => import('../views/dashboard/welcome')
},

二级菜单


{
  icon: 'el-icon-edit-outline',
  label: '表单页',
  showInMenu: true,
  path: '/dashboard/form',
  auth: ['admin', 'customer'],
  children: [
    {
      path: '/dashboard/basic-form',
      label: '基础表单',
      showInMenu: true,
      auth: ['admin', 'customer'],
      name: 'BasicForm',
      component: () => import('../views/dashboard/basicForm')
    },
    {
      path: '/dashboard/step-form',
      label: '分布表单',
      showInMenu: true,
      auth: ['admin'],
      name: 'StepForm',
      component: () => import('../views/dashboard/stepForm')
    }
  ]
},
