布局

页面整体布局是一个产品最外层的框架结构，往往会包含导航、页脚、侧边栏、通知栏以及内容等。在页面之中，也有很多区块的布局结构。在真实项目中，页面布局通常统领整个应用的界面，有非常重要的作用。

Elite 的布局

在 elite 中，我们抽离了使用过程中的通用布局并称作“场景”，都放在 scene 目录中。

  • Dashboard：后台页面布局，包含了头部、侧边栏、面包屑和通知栏等

    elite 采用的是四块布局方案。这种布局方案符合一般人对于后台的认知程度。

    logo
    面包屑
    用户信息和快捷菜单
    菜单
    主界面

  • Default：空白布局，此布局是用户进入 elite 以后见到的第一个布局页面，默认情况下为一张底图并保持一个加载中的状态。

    此处可以用来加载和判断用户登录状态等。

自定义布局

elite 支持自定义布局。您只需要在 layouts 目录下建立一个布局的文件夹，在对应的 html 结构里保留一个 <router-view /> 标签即可完成布局页面的设定

同时，您需要在路由中引入您的文件，并且将其挂载到对应位置。例如：


import MyLayout from '@/layouts/myLayout'

const Routes = [
  // ... 一些代码
  {
    path: '/my-layout',
    name: 'MyLayout',
    showInMenu: false,
    component: MyLayout,
    children: [
      // ... 一些代码
    ]
  },
  // ... 一些代码
]
